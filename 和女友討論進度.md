# 怎麼讓女友看到進度、一起討論要改什麼

照下面做，她就有一個「網址」可以玩外掛，還有一個「進度頁」可以看做了什麼、要改什麼。

---

## 一、部署到一個「她看得到的網址」

這樣她不用裝程式，開瀏覽器就能玩、能測。

1. **主站部署到 Vercel**（若還沒做）
   - 程式先 push 到 GitHub（你之前做過）。
   - 到 https://vercel.com 用 GitHub 登入 → **Add New…** → **Project** → 選你的 repo（例如 `sooty-game`）→ **Deploy**。
   - 部署好會給你一個網址，例如：`https://sooty-game-xxx.vercel.app`。
   - **把這個網址傳給女友**，她開這個網址就能玩小黑炭。

2. **之後你每次改完、push 上 GitHub**
   - Vercel 會自動重新部署，她重新整理同一個網址就會看到最新版。
   - 不用再傳新網址。

---

## 二、讓她看到「每一個製作、修改的進度」

有兩個地方可以看進度，你兩個都給她連結就好。

### 方式 A：專案裡的「進度」檔案（推薦，方便討論）

- 專案裡有一份 **`進度.md`**，裡面有：
  - **已完成**：目前做好的功能
  - **進行中 / 待部署**：接下來要完成的事
  - **待討論 / 想修改的**：你們可以一起列「想加、想改」的點，討論完再去做

**她怎麼看：**

- **選項 1**：她打開你給的**主站網址**（Vercel 那個），捲到最下面，點 **「開發進度（與女友討論用）」**，就會連到 GitHub 上的 `進度.md`。
- **選項 2**：你直接把進度頁的網址傳給她（把下面換成你的 GitHub 帳號與 repo 名稱）：
  ```
  https://github.com/turtletotamus/sooty-game/blob/main/進度.md
  ```

**你怎麼更新：**

- 用 Cursor / VS Code 打開專案裡的 **`進度.md`**，改「已完成」「待討論」等內容，存檔。
- 在終端機執行：
  ```bash
  git add 進度.md
  git commit -m "更新進度"
  git push origin main
  ```
- 她重新整理進度頁（或再點一次「開發進度」連結），就會看到最新內容。

這樣你們就有一個「共同進度表」可以討論還有什麼要修改。

### 方式 B：GitHub 上的 commit 紀錄

- 你每次 `git commit` + `git push`，都會在 GitHub 上留下一筆紀錄。
- 她若會看 GitHub，可以打開你的 repo：
  ```
  https://github.com/turtletotamus/sooty-game/commits/main
  ```
- 每一筆 commit 的訊息就是你當次「做了什麼」的簡短說明；點進去還可以看到改了哪些檔案。

---

## 三、總結：給女友的兩個連結

部署好之後，你只要傳給她：

| 連結 | 用途 |
|------|------|
| **主站網址**（例如 `https://sooty-game-xxx.vercel.app`） | 玩小黑炭、測功能，你 push 後她重整就看到最新版 |
| **進度頁**（例如 `https://github.com/turtletotamus/sooty-game/blob/main/進度.md`） | 看「已完成 / 待討論 / 想修改」的進度，方便討論還有什麼要改 |

你只要改 `進度.md` 並 push，她就能看到每一個製作、修改的進度，兩人比較好討論。
