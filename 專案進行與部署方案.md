# 小黑炭專案 － 進行方式與部署方案

目標：**女友可以打開一個網址就看到「實際長怎樣」**，不是只看文字。  
這份說明：專案怎麼進行、怎麼部署、她怎麼看。

---

## 一、整體怎麼進行

| 項目 | 說明 |
|------|------|
| **程式在哪** | 你電腦裡的這個專案（sooty-game），用 Git 推到 GitHub，所以有一份在網路上。 |
| **誰改程式** | 你（或之後加女友為協作者）在 Cursor 改、commit、push。 |
| **「實際長怎樣」** | 把這份專案**部署到 Vercel**，會得到一個**網址**。任何人開那個網址 = 打開你們的網站，和你在本機 `npm run dev` 看到的畫面一樣。 |
| **女友怎麼看** | 你部署好後**把網址傳給她**。她開瀏覽器點進去 = 看到實際的網站（餵食、小黑炭、番茄鐘、語言切換等），不是文件。 |
| **之後你改版** | 你改程式 → push 上 GitHub → Vercel 會自動重新部署 → 女友重新整理同一個網址就看到新版本。 |

所以：**部署一次 = 有一個「線上版」網址；女友看的就是那個網址的實際畫面。**

---

## 二、部署到哪裡、怎麼做（讓女友看到實際長相）

用 **Vercel** 部署，不用自己管主機，且和 Next.js 相容最好。

### 方式 A：在網頁上點一點就部署（推薦，不用打指令）

1. **確認程式已在 GitHub**
   - 你之前已經 push 到 `https://github.com/turtletotamus/sooty-game`（若還沒，先做完 push）。

2. **登入 Vercel**
   - 打開：**https://vercel.com**
   - 點 **Sign Up** 或 **Log In**，選 **Continue with GitHub**，用你的 GitHub 帳號授權。

3. **從 GitHub 匯入專案**
   - 登入後點 **Add New…** → **Project**。
   - 畫面上會列出你的 GitHub repo，找到 **sooty-game**，右邊點 **Import**。
   - **Framework Preset** 保持 **Next.js**（通常會自動偵測）。
   - **Root Directory** 維持空白。
   - 直接點 **Deploy**。

4. **等一兩分鐘**
   - 跑完會顯示 **Congratulations!** 和一個網址，例如：
     `https://sooty-game-xxxx.vercel.app`
   - 這個網址就是**線上版**，和本機跑起來的畫面一樣。

5. **給女友**
   - 把這個網址傳給她（例如傳 Line）。
   - 她用手機或電腦瀏覽器打開，就能看到**實際的網站**（小黑炭、按鈕、語言切換等），不是只有文字。

之後你改程式、push 到 GitHub，Vercel 會自動再部署，女友**同一個網址**重新整理就會看到最新版。

### 方式 B：用指令部署（進階）

若你已安裝 Vercel CLI 且登入過，在專案根目錄執行：

```bash
cd /Users/turtletotamus/sooty-game
npx vercel --prod
```

依提示選或按 Enter，最後也會給你一個網址，效果同上。

## 三、部署完成後：你與女友各做什麼

| 角色 | 要做的事 |
|------|----------|
| **你** | 1) 把 Vercel 給的網址傳給女友。 2) 之後改程式就照常 commit + push，Vercel 會自動更新。 3) 若要討論「接下來改什麼」，可一起看專案裡的 `進度.md`。 |
| **女友** | 1) 打開你傳的網址，看實際畫面、實際玩。 2) 想改什麼、想加什麼跟你說，或寫在 `進度.md` 的「待討論」裡。 |

她**不需要**裝 Node、Git、Cursor，只要會開瀏覽器即可。

---

## 四、若 push 到 GitHub 還沒成功

部署前一定要先讓程式在 GitHub 上。若你之前 `git push` 有 403 或沒成功：

1. 再試一次：**清掉 Mac 存的 GitHub 密碼**，用 **Personal Access Token** 當密碼推一次（之前有寫過步驟）。
2. 或改用 **GitHub Desktop**：安裝後登入 GitHub，把 sooty-game 資料夾加進去，用介面按 **Push**，就不用打指令。

程式一成功 push 到 GitHub，就可以立刻到 Vercel 用「方式 A」匯入並 Deploy。

## 五、一句話總結

- **專案進行**：你在 Cursor 改專案 → push 到 GitHub。  
- **部署**：在 Vercel 用 GitHub 匯入 sooty-game → 按 Deploy → 得到一個網址。  
- **女友看實際長怎樣**：開你給的那個網址，就是整個專案「實際長怎樣」的畫面。
