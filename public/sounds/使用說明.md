# 🎵 自訂音檔使用說明

## 📂 如何放置檔案

### 方法 1：使用 Finder（Mac）或檔案總管（Windows）

1. **找到專案資料夾**
   - 路徑：`/Users/turtletotamus/sooty-game/public/sounds/`

2. **將音檔複製到這個資料夾**
   - 直接拖放檔案到 `public/sounds/` 資料夾
   - 或複製檔案後貼上到這個資料夾

### 方法 2：使用終端機

```bash
# 進入專案資料夾
cd /Users/turtletotamus/sooty-game

# 將音檔複製到 sounds 資料夾（替換 your-file.mp3 為你的檔案名稱）
cp /path/to/your-file.mp3 public/sounds/alarm-gentle.mp3
```

## 📝 檔案命名規則（重要！）

系統會自動偵測以下名稱的檔案：

### 提醒聲（Alarm Sounds）
- `alarm-gentle.mp3` - 溫和提醒聲
- `alarm-bell.mp3` - 鈴聲
- `alarm-chime.mp3` - 鐘聲
- `alarm-beep.mp3` - 嗶嗶聲

### 背景音效（Background Sounds）
- `tick.mp3` - 滴答聲（預設背景音效）
- `fire.mp3` - 火爐聲
- `rain.mp3` - 下雨聲

## ✅ 使用步驟

1. **準備你的音檔**
   - 確保音檔格式為 `.mp3`, `.wav`, 或 `.ogg`
   - 將檔案重新命名為上述規則中的名稱

2. **放置檔案**
   - 將檔案放到 `public/sounds/` 資料夾

3. **重新啟動開發伺服器**
   ```bash
   # 如果正在運行，先停止（Ctrl+C）
   # 然後重新啟動
   npm run dev
   ```

4. **系統會自動偵測**
   - 打開番茄鐘設定
   - 系統會自動使用你放置的音檔
   - 如果檔案不存在，會使用預設的生成音效

## 💡 提示

- ✅ 檔案名稱必須**完全符合**命名規則（大小寫敏感）
- ✅ 支援多種格式：`.mp3`, `.wav`, `.ogg` 等
- ✅ 如果某個音檔不存在，系統會自動使用預設的生成音效
- ✅ 背景音效會自動循環播放
- ✅ 提醒聲只會播放一次

## 🔍 檢查檔案是否正確放置

打開終端機執行：

```bash
cd /Users/turtletotamus/sooty-game
ls -la public/sounds/
```

你應該會看到你放置的檔案列表。

## 📋 範例

假設你有以下音檔：
- `my-alarm.mp3` → 重新命名為 `alarm-gentle.mp3`
- `ticking.mp3` → 重新命名為 `tick.mp3`
- `fireplace.mp3` → 重新命名為 `fire.mp3`

然後將它們放到 `public/sounds/` 資料夾即可！
